Quels sont les raisons pour lesquelles vous pouvez adopter Tendoo ?
Tout d'abord deux personnes sont concernées par Tendoo : les non développeurs et les développeurs. 
Pour les premiers, Tendoo est fourni avec l'essentiel pour faire un blog complet avec prise en charge de plusieurs fonctionnalités avancées, Il est notamment possible :

- Gérer les publications (écrire un article, modifier, supprimer)
- Gérer les commentaires (Supprimer, valider)
- Autoriser les commentaires
- Autoriser les utilisateurs non inscrit à poster des commentaires
- Créer et gérer des privilèges
- Accorder des actions aux privilèges
- Créer et autoriser la création d'utilisateurs
- Espace membre (message, modification des infos. personnelles)
- Gérer les contacts (Réponse immédiate disponible pour les membres inscrit)
- Gérer la page d'accueil en choisissant la quantité et la qualité des informations, ainsi que la manière dont ces informations se présentent.
- Gérer les informations système du site (nom, logo, format de l'heure, Afficher les icônes des applications à l'accueil de l'espace administration,le décalage horaire)
- Afficher les statistiques sur les visites, ce qui vous permet de mieux gérer votre traffic et l'évolution de votre site web
- Restaurer le site web, supprime tous les fichiers installés par les modules & thème et re-installe le site web.
- Connexion au Store qui pourra être désactivé dans la version 0.9.6
- Possibilité d'ouvrir les modules directement depuis la page d'accueil, lorsque les icones ont été activé dans les paramètres.

Il s'agit bien sur la des fonctionnalités simplifiées et les plus utilisées par les non développeurs.

Tendoo est également un OUTIL de développement pour les développeurs dans la mesure où ils offre des utilitaires que ce soit en classe et méthode au développeur, qui n'aura plus à ce soucier de faire :

- Un espace membre Tendoo fournie une API dessus
- Un espace administration Tendoo fourni également une API dessus.
- Un système de connexion à la base de donnée Tendoo grâce à sa méthode db renvoi la ressource mysql qui peut être utilisé dans les thème ou les modules.
- Faire un style pour l'espace administration, qui utilise actuellement bootstrap.

L'élément le plus important qu'il faut souligner, c'est qu'il y a une véritable indépendance entre les modules et le système. Et pour cause tous les modules sont installés ULTERIEUREMENT à l'installation du cms, c'est grâce à cet aspect qu'il est possible de développer un site web en marge du cms.
Les développeurs d'application auront un avantage certains en utilisant Tendoo. Un étude sur le temps de création de module à été effectué et il s'est avéré que :

- Il est possible de faire un thème (professionnel, sans interface embarqué) en 12 heures
- Il est possible de faire un module basique tels que "Tendoo Contact Manager" en 24 heures.

Nous travaillons sur les fonctionnalités du système afin de réduire considérablement le temps de travail afin de rendre l'utilisation d'Tendoo plus productive. Alors n'hésitez pas à tester Tendoo sur http://github.com/Blair2004/Tendoo-cms
